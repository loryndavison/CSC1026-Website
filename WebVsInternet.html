Web vs Internet

There has always been confusion over the differences between the internet and the world wide web. The following quotes from Tim Berners Lee make this a bit clearer.

"The Web is an abstract (imaginary) space of information ... On the Net, the connections are cables between computers; on the Web, connections are hypertext links ... The Web made the net useful because people are really interested in information"


- Tim Berners Lee

The Internet ('Net) is a network of networks. Basically it is made from computers and cables...If you put the right address on a packet, and gave it to any computer which is connected as part of the Net, each computer would figure out which cable to send it down next so that it would get to its destination. That's what the Internet does. It delivers packets - anywhere in the world, normally well under a second.


- Tim Berners Lee


How the Internet works

What is the internet?

  • A networking infrastructure that allows hardware devices to connect to one another.
  • Run using protocols which specify how transactions are to be carried out.
  • The use of software clients and servers allow users and devices to access and use internet services.

Clients, Servers and Peers

  • The most common internet services are based on a client/server model using appropriate protocols.
  • Local software clients usually access centralised remote servers. (eg. Email via Outlook server)
  • Peer-to-Peer technology can directly connect multiple decentralised computers where each acts as both client and server. (eg. VOIP/video calling)

Protocols

  • Clients and servers typically use a stack of different protocols to provide service.
  • The most common protocols in use today are those associated with web and email services. (eg. HTTP/HTTPS, SMTP/IMAP/POP)
  • Below is a table showing some of the most common protocols:
Protocol Notes
HTTP HyperText Transfer Protocol Core protocol for web transactions
SSL Secure Sockets Layer Used for data encryption
HTTPS Secure HyperText Transfer Protocol A combination of SSL and HTTP
SMTP/POP/IMAP Simple Mail Transfer Protocol/Post Office Protocol/Internet Message Access Protocol Used in combination to provide most email services
FTP File Transfer Protocol Used to exchange files between computers
SSH and SFTP Secure SHell and Secure FTP Encrypted access to remote computers and an extension for encrypted file transfer
BitTorrent Protocol for peer-to-peer data access and exchange

IP: Routing Packets

  • Internet protocol routes packets based on the address information in their header
  • IP routing is connection-less. This means:
    • It doesn't require a fixed connection between the endpoints.
    • It enables data to travel via multiple routes.
    • One wire can handle multiple connections.
    • Packets can be re-routed around points of failure within the system.
  • However risks to IP routing include:
    • Corruption of data
    • Loss of packets
    • Duplicate packets arriving
    • Packets being delivered out of sequence
  • Some of the protocols included in IP are:
    • BGP (Border Gateway Protocol)
    • IS-IS (Intermediate System - Intermediate System)
    • OSPF (Open Shortest Path First)
    • RIP (Routing Information Protocol)
  • IP only promises best-effort delivery

TCP/IP

  • TCP (Transmission Control Protocol)
    • Handles direct connection on computers between client/server
    • Breaks outgoing messages into packets to send via IP
    • Assembles incoming packers into messages to pass back to the application
  • Packets have two parts to them:
    • Header (contains information on addressing, sequencing and metadata)
    • Body (contains actual data payload)
  • IP (Internet Protocol)
    • Handles delivery of the packets to/from any device with an IP address
    • IP address is a unique numeric identifier for a device on the internet

What is meant by best effort delivery?

Performance of IP routing can be affected by various factors.

  • Unpredictability means that IP cannot guarantee that all packets..
    • will arrive within a fixed time
    • will arrive in the correct order
    • will arrive at all
  • To help resolve these issues, TCP on the destination device checks and reassembles packets based on header information and can..
    • request re-transmission of lost/corrupted packets
    • correctly re-sequence packets
    • refuse to accept/wait for any more packets(drop connection)

Domain Names

  • Domain names make the internet easier to use by mapping numeric IP addresses to text based names.
  • When a device makes a request using a domain name it is resolved back into a numeric IP address by a DNS server
