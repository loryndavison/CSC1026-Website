Web Clients and Servers

Web Clients

  • A web client is a piece of software to create and send HTTP requests and handle their responses.
  • Mosasic kick-started mainstream interest followed by internet explorer and netscape.
  • "Under the hood" many share common layout engines (eg. Trident, Gecko, KHTML/Webkit etc)
Feature Description
Request generation Assemble URIs and initiate the request(s)
Response Handling Accept server response codes and act accordingly
Content Parsing Read/process HTML markup and CSS from response
Maintain state Create and manage storage of local tokens and cookies
Client-side scripting Run JavaScript supplied by server response
Encryption Manage SSL encrypted transactions (over HTTPS)
Media Objects Display/play embedded media objects

Web Servers

  • The term web server can refer to hardware or software.
    • In terms of hardware, a web server is a computer that stores a website's component files (e.g. HTML documents, images, CSS stylesheets, and JavaScript files) and delivers them to the end-user's device.
    • In terms of software, a web server includes several parts that control how web users access hosted files. The server listens on an open network port for incoming HTTP requests and responds accordingly.
Feature Description
Path Translation Locate resource specified in the URL
Virtual Hosting Run multiple websites from one server
Access Restriction Control who can access the resources
Log Access and Errors Record whats going on and what went wrong
Encrypted Operation Protect sensitive data in transit
Load balancing Manage traffic and processing resources
Server Scripting Run external programs and applications
